# Content Analysis: 提示词模板.md

## Highlights & Key Insights

- 模板强调研究对象必须是具备行动能力、权限、工具、记忆和协作能力的 AI Agent，而非泛化 AI 安全。
- 重点要求所有关键判断绑定权威证据、发布日期、适用范围和可信度，避免编造引用。
- 输出目标明确指向组织决策、工程落地、风险治理和安全建设，而非概念综述。
- 报告必须同时覆盖场景、架构、威胁、防御、评估、路线图和 Build / Buy / Integrate 取舍。
- 强调不给出可被直接滥用的攻击代码或绕过细节。

## Structure Assessment

- Current flow: 采用“研究边界 → 资料源要求 → 研究目标 → 分析框架 → 实战场景 → 研究方法 → 报告结构 → 输出风格 → 自检标准”的顺序，逻辑完整，但缺少统一的 Markdown 章节层级与 frontmatter。
- Suggested sections:
  - 一、研究边界：定义研究对象与排除项，是全文约束条件。
  - 二、资料源要求：规定证据优先级与引用标准，约束研究方法。
  - 三、研究目标：明确要回答的核心问题，定义产出价值。
  - 四、分析框架：给出分析维度，避免研究只停留在单一安全层。
  - 五、必须覆盖的实战问题：把抽象研究落到 3-5 个具体 Agent 场景。
  - 六、研究方法：说明证据、建模、防御评估与结论生成流程。
  - 七、报告结构：提供最终报告模板骨架。
  - 八、输出风格：约束表达密度与禁区。
  - 九、自检标准：定义交付前的质量校验清单。

## Reader-Important Information

- “研究重点不是‘模型会不会回答错’”是全文最关键的立场，适合高亮。
- 资料优先级、核心问题、实战场景输出项、研究方法 10 步、自检 10 项都适合保留为显式列表。
- 报告结构中的表格字段和强制覆盖项是实际落地时最常被引用的部分，应保证可快速定位。
- “禁止”项与“不建议清单”属于使用边界，适合强调。

## Formatting Issues

- 开头缺少 frontmatter，无法直接沉淀 title、slug、summary 和 description。
- 顶层章节使用中文序号文本而非统一 Markdown 标题，不利于目录和跳转。
- 关键立场句埋在正文中，可读性不足。
- 若干过渡句如“资料优先级如下：”“请回答以下核心问题：”“按以下流程完成研究：”更适合作为强调行，帮助扫描。
- 整体已经存在列表和表格骨架，无需改写内容，只需补足层级与强调。
